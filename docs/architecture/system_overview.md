# システム概要

## 目的

竿の振動（アタリ）を検知して自動でリールを巻き上げ、巻き上げ完了時に釣り人に扮した
LLMキャラクターが応答する全自動ワカサギ釣りマシーン。IoT（センサー・アクチュエータ制御）と
AI（LLMによるキャラクター応答生成）を組み合わせた学習用システム。

## コンポーネント構成

```
                          ┌─────────────────────────────┐
                          │      Raspberry Pi 4          │
                          │                               │
  振動センサー ─────────▶│  device/wakasagi_device      │
  フォトスイッチ ───────▶│  ├─ sensors/                 │
                          │  ├─ state_machine/ (transitions)
                          │  ├─ actuators/ ──────────────┼──▶ モーター(リール)
                          │  └─ output/                   │
                          │        │                       │
                          └────────┼───────────────────────┘
                                   │ invoke_model()
                                   ▼
                          ┌─────────────────────┐
                          │  cloud/bedrock       │
                          │  AWS Bedrock Runtime │
                          │  (プロンプトベース)   │
                          └─────────────────────┘
```

- `device/` と `cloud/` はPythonパッケージとして分離。現状はどちらもPi上の同一プロセスから
  呼び出すが、`cloud/bedrock` を独立させておくことで将来Lambda化する際に移植しやすくする。
- `edge/` は将来のAWS IoT Greengrass化・ローカルLLM化のための置き場所（現状は構想メモのみ）。

## データフロー（1サイクル）

1. 振動センサーが竿の振動を検知（GPIO入力）
2. 状態機械がデバウンス判定を行い、誤検知でなければ巻き上げを開始
3. モーターを駆動し、フォトスイッチが満巻き位置を検知したら停止
4. `cloud/bedrock/client.py` 経由でBedrockにプロンプト（キャラクター人格＋巻き上げ結果）を送信
5. 生成された応答をテキスト出力（将来はTTSで音声出力）
6. 状態機械が待機状態に復帰

詳細な状態遷移は [state_machine.md](state_machine.md) を参照。
